# 卡牌描述数字与 BaseLib 变量接口说明

## 核心规则

卡牌描述中的可变数字不要直接写死在本地化文本里。只要这个数字可能因为升级、力量、敏捷、虚弱、易伤、目标状态、当前战斗状态等原因变化，就应该同时满足两件事：

1. 在卡牌类的 `CanonicalVars` 中声明对应的 `DynamicVar`。
2. 在 `localization/*/cards.json` 的描述文本中引用同名变量。

也就是说，代码和文本必须引用同一个变量名。只在结算代码里改数字，但描述里写死 `1`、`2`、`60`，卡面就不会跟着升级或状态变化。

## 伤害和格挡

伤害和格挡不要优先使用普通 `DamageVar` / `BlockVar`。普通变量能参与结算，但卡面预览不一定正确显示力量、敏捷、虚弱、易伤等战斗修正。

推荐用 BaseLib 的 calculated var：

```csharp
protected override IEnumerable<DynamicVar> CanonicalVars => [
    ..MakeCalculatedDamage(6, static (_, _) => 0m)
];
```

文本中引用：

```json
"description": "造成 {CalculatedDamage} 点伤害。"
```

格挡同理：

```csharp
protected override IEnumerable<DynamicVar> CanonicalVars => [
    ..MakeCalculatedBlock(5, static (_, _) => 0m)
];
```

文本中引用：

```json
"description": "获得 {CalculatedBlock} 点格挡。"
```

如果一张牌同时有伤害和格挡，不要同时用默认 `CalculatedDamage` / `CalculatedBlock`，因为它们会生成默认的基础变量，容易互相冲突。应使用自定义名字：

```csharp
protected override IEnumerable<DynamicVar> CanonicalVars => [
    ..MakeCalculatedDamage("BladeDamage", 17, static (_, _) => 0m),
    ..MakeCalculatedBlock("BladeBlock", 17, static (_, _) => 0m)
];
```

文本中引用：

```json
"description": "造成 {BladeDamage} 点伤害。\\n获得 {BladeBlock} 点格挡。"
```

升级时也要升级对应基础变量：

```csharp
DynamicVars["BladeDamageBase"].UpgradeValueBy(3m);
DynamicVars["BladeBlockBase"].UpgradeValueBy(3m);
```

默认 calculated damage/block 的基础值变量名是 `CalculationBase`。例如：

```csharp
DynamicVars.CalculationBase.UpgradeValueBy(3m);
```

## 普通数值

抽牌数、选择数量、能量、百分比、阈值、持续次数等普通数字，用 `DynamicVar`：

```csharp
protected override IEnumerable<DynamicVar> CanonicalVars => [
    new DynamicVar("Draw", 1m),
    new DynamicVar("Advance", 1m)
];
```

文本：

```json
"description": "抽 {Draw} 张牌。\\n当前祷告计时加快 {Advance} 回合。"
```

结算也应该读取同一个变量：

```csharp
await CardPileCmd.Draw(choiceContext, (int)DynamicVars["Draw"].BaseValue, Owner);
await PrayerManager.Accelerate(choiceContext, Owner.Creature, (int)DynamicVars["Advance"].BaseValue);
```

升级：

```csharp
DynamicVars["Draw"].UpgradeValueBy(1m);
```

## Power 数值

给 buff/debuff 的层数，优先使用 `PowerVar<TPower>`：

```csharp
protected override IEnumerable<DynamicVar> CanonicalVars => [
    new PowerVar<WeakPower>(2m)
];
```

文本引用 Power 类名：

```json
"description": "给予 {WeakPower} 层虚弱。"
```

结算也读同一个变量：

```csharp
await PowerCmd.Apply<WeakPower>(
    choiceContext,
    target,
    DynamicVars[typeof(WeakPower).Name].BaseValue,
    Owner.Creature,
    this);
```

升级：

```csharp
DynamicVars[typeof(WeakPower).Name].UpgradeValueBy(1m);
```

## 不建议的写法

不要这样写：

```json
"description": "抽 2 张牌。"
```

同时代码里：

```csharp
await CardPileCmd.Draw(choiceContext, IsUpgraded ? 3 : 2, Owner);
```

这种实现虽然能结算，但升级后卡面仍然显示旧数字。应改成 `DynamicVar("Draw", 2m)`，文本写 `{Draw}`，升级时 `UpgradeValueBy(1m)`。

## 本项目目前的例外

祷告牌的描述和选项由 `NunPrayerCard` 额外生成，涉及不同祷告档位、祷告区悬浮描述和延迟触发效果。它们不能简单按普通卡牌规则批量替换，需要单独维护。
