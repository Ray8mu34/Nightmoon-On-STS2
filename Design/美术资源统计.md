# 比例说明

### 基础资源

1. 能量图标

自定义人物通常会拥有自己的卡牌池、遗物池和药水池，因此需要准备人物专属能量图标。能量图标分为小图标和大图标，两者用途不同，不能只准备其中一个。

| 资源    |    尺寸 | 透明背景 | 用途                        | 是否必须 |
| ----- | ----: | ---- | ------------------------- | ---- |
| 小能量图标 | 24×24 | 是    | 显示在卡牌描述文本中，例如能量符号         | 必须   |
| 大能量图标 | 74×74 | 是    | 显示在卡牌费用、tooltip 或较大的 UI 中 | 必须   |

小能量图标必须在很小的尺寸下保持清楚。图形不应依赖细线、复杂纹样或小字。推荐使用轮廓明确的符号，例如圣环、烛火、血滴、裂痕、十字或其他与职业机制相关的图形。

大能量图标可以保留更多材质、光效和装饰，但主体轮廓仍应清晰。小图标和大图标应使用同一视觉母题，避免看起来像两套不同资源。

2. 卡牌插画

每张自定义卡牌都应准备卡图。刚开始制作时，可以先使用占位图，但正式测试前应尽量为每张卡准备独立插画。

| 资源     |    推荐尺寸 |      比例 | 透明背景  | 用途                     |
| ------ | ------: | ------: | ----- | ---------------------- |
| 普通卡牌卡图 | 250×190 |   25:19 | 通常不需要 | 攻击、技能、能力牌的中间插画         |
| 先古卡图   | 250×351 | 250:351 | 通常不需要 | Ancient card 使用的竖向特殊插画 |

普通卡牌卡图只需要绘制插画本身，不需要绘制卡框、费用、标题、描述文字或类型栏。卡框和文字由游戏 UI 负责显示。

卡图应避免把主体画得过小。卡牌在游戏中显示面积有限，复杂背景、细碎人物、过多文字都会降低识别度。推荐采用明确的主体剪影和较强的明暗关系。

3. 自定义遗物图标

只要人物 Mod 中包含自定义遗物，就需要准备遗物图标。遗物资源通常包括小图标、轮廓图和大图。

| 资源    |      尺寸 | 透明背景 | 用途                | 是否必须      |
| ----- | ------: | ---- | ----------------- | --------- |
| 遗物小图标 |   85×85 | 是    | 遗物栏、奖励界面中显示       | 有自定义遗物时必须 |
| 遗物轮廓  |   85×85 | 是    | 遗物未识别、轮廓或特殊 UI 显示 | 建议        |
| 遗物大图  | 256×256 | 是    | 商店、详情、奖励界面中的大图    | 建议        |

遗物小图标必须在 85×85 尺寸下保持主体明确。外轮廓应清楚，边缘应留出透明安全区，避免图标贴边。遗物大图可以包含更多材质、裂纹、金属、布料、烛光、血迹等细节，但构图应与小图标保持一致。

4. 自定义药水图标

只要人物 Mod 中包含自定义药水，就需要准备药水主图。药水轮廓图不是绝对必须，但建议同时准备，便于 hover、选中或特殊 UI 状态使用。

| 资源   |    推荐尺寸 |  比例 | 透明背景 | 用途            |
| ---- | ------: | --: | ---- | ------------- |
| 药水主图 | 256×256 | 1:1 | 是    | 药水本体图像        |
| 药水轮廓 | 256×256 | 1:1 | 是    | hover、选中或轮廓效果 |

药水主图和药水轮廓应使用相同画布、相同位置和相同缩放。轮廓图不能重新构图，否则 UI 状态切换时会产生错位。

药水图标应优先保证瓶身轮廓清楚。瓶塞、液面、标签、光效可以作为辅助细节，但不应让图标在小尺寸下变成一团复杂纹理。

5. 自定义 Power / Buff 图标

人物机制通常会通过 Power 或 Buff 显示在角色状态栏中。只要 Mod 中包含自定义状态，就应准备对应图标。

| 资源        |      尺寸 |  比例 | 透明背景 | 用途            |
| --------- | ------: | --: | ---- | ------------- |
| Power 小图标 |   64×64 | 1:1 | 是    | 状态栏显示         |
| Power 大图标 | 256×256 | 1:1 | 是    | tooltip 或详情显示 |


Power 图标应尽量使用明确的符号，而不是复杂场景。一个好的 Power 图标通常只表达一个概念，例如“忏悔”“祷告”“赦免”“裂伤”“祝福”“延迟触发”“自伤结算”等。

如果同一机制拥有多个阶段，应保持图标系统统一。例如忏悔类图标可以使用暗红、瘀紫、干血色、裂痕、荆棘、倒刺十字；祷告类图标可以使用象牙白、旧金、烛光黄、圣环、蜡烛、合掌、祷文等元素。

6. 战斗人物视觉

战斗人物视觉用于在战斗中显示角色本体。第一次制作时可以先使用占位角色或简单静态图；正式制作时应准备独立的战斗人物视觉。

| 资源           | 尺寸 / 比例 | 透明背景 | 用途        | 是否必须     |
| ------------ | ------- | ---- | --------- | -------- |
| 战斗人物图或战斗人物场景 | 不固定     | 通常需要 | 战斗中显示角色本体 | 不使用占位时必须 |

战斗人物不要求固定比例。常见做法是使用偏竖向的完整站姿，使角色在战斗场景中容易识别。角色通常应面朝右侧，以适配玩家角色在战斗画面中的方向。

战斗人物可以是静态 PNG，也可以是 Godot 场景。静态图适合第一次制作人物 Mod；Godot 场景适合后续加入动画、挂点、表情、特效或复杂层级。

战斗人物图应满足以下要求：

| 要求     | 说明                   |
| ------ | -------------------- |
| 透明背景   | 角色应独立于背景，方便放入战斗场景    |
| 剪影清楚   | 小尺寸下仍能看出角色姿态和武器 / 道具 |
| 面朝右侧   | 符合玩家角色在战斗中的默认朝向      |
| 重心稳定   | 避免角色站姿过斜或视觉重心偏移      |
| 留出边缘空间 | 不要让武器、披风、光效贴到画布边缘    |


### 正式人物资源

1. 角色选择图标

角色选择图标用于人物选择界面中的小图标显示。推荐使用 1:1 画布。

| 资源      | 推荐比例 |          推荐尺寸 | 透明背景 | 用途         |
| ------- | ---: | ------------: | ---- | ---------- |
| 角色选择小图标 |  1:1 | 128×128 或更高源图 | 是    | 人物选择界面的小图标 |

图标不应直接缩小完整立绘。完整人物缩成小图后通常会丢失识别度。更合适的做法是截取头像、武器、圣物、面具、手势或机制符号作为图标主体。

2. 人物头像 / 顶栏图标

人物头像可用于部分 UI 显示，也可以作为人物识别资源的一部分。推荐使用 1:1 画布。

| 资源   | 推荐比例 | 透明背景 | 用途             |
| ---- | ---: | ---- | -------------- |
| 人物头像 |  1:1 | 是或可选 | 头像、顶栏或其他 UI 显示 |

头像应突出面部、头饰、面具或职业标志。背景可以透明，也可以使用简单底色，但应避免复杂背景干扰人物识别。

3. 地图标记

地图标记用于地图界面中表示当前角色或相关 UI 标识。推荐使用 1:1 图标。

| 资源   | 推荐比例 |       推荐尺寸 | 透明背景 | 用途                  |
| ---- | ---: | ---------: | ---- | ------------------- |
| 地图标记 |  1:1 | 128×128 源图 | 是    | 地图上的角色标记，也可兼作小型机制图标 |

地图标记应比头像更符号化。它通常不是完整角色图，而是一个可以快速识别的职业标志。

4. 角色选择背景

角色选择背景不应简单理解为一张固定比例的“大图”。在 STS2 中，更准确的做法是制作角色选择背景场景。该场景可以包含背景、角色立绘、装饰层、光效或简单动画。

| 资源       | 推荐形式                  | 用途          |
| -------- | --------------------- | ----------- |
| 角色选择背景场景 | Godot Control 场景或组合图层 | 角色选择界面的展示背景 |

如果使用静态图作为背景，可以先按竖向构图绘制，例如 2:3 或接近角色立绘海报的比例。但这只是美术构图建议，不是固定技术要求。

角色选择背景应表现角色的主题氛围。例如忏悔主题可以使用暗色教堂、血迹、荆棘、裂纹、刑具、阴影；祷告主题可以使用烛光、圣环、旧金、经文、祭坛、柔和光束。
